Question-1: Write down Differences Between var, let and const in JavaScript.

Answer:

var:

var declarations are globally scoped.

It can be updated and re-declared.

It can be declared without initialization.


let:

var declarations are block scoped.

It can be updated but cannot be re-declared.

It can be declared without initialization.


const:

var declarations are block scoped.

It cannot be updated or re-declared.

It can't be declared without initialization.

Question-2: What are the Differences Between Regular function and Arrow funtion in JavaScript.

Answer:

Arrow function is a new feature introduced in ES6 that is a more concise syntax for writing function expressions. While both regular JavaScript functions and arrow functions work in a similar manner, there are certain differences between them.

Inside of a regular JavaScript function, this value (aka the execution context) is dynamic.The regular function can easily construct objects.
But the behavior of this inside of an arrow function differs considerably from the regular function's this behavior. The arrow function doesn't define its own execution context.

Inside the body of a regular function, arguments is a special array-like object containing the list of arguments with which the function has been invoked. On the other side, no arguments special keyword is defined inside an arrow function.

Question-3: What are the Differences Between map(), forEach() and filter(), find() in JavaScript.

Answer:

Differences between map() and forEach():

The main difference between map() and forEach() is that the map method returns a new array by applying the callback function on each element of an array, while the forEach method doesn't return anything. We can use the forEach() method to mutate the source array, but this isn't really the way it's meant to be used. Instead, it's great for when we need to do some action with each element of the array. On the other hand, the map() method is used for creating a new array, and thus, it's chainable. We can call a number of map operations one after the other. The forEach method doesn't return anything, so we can't chain it with any other methods - it's not chainable.

The find() method is used to find all the descendant elements of the selected element. It finds the element in the DOM tree by traversing through the root to leaf. The filter() method is used to filters all the elements and returns the element that matches and the element that do not match are removed. The only difference is the filter() method search through all the elements while find() method search through all the child elements only.

Question-4: Why we use Template String?

Answer:

Some benefits of using template string:

1. Template Literals use back-ticks (``) rather than the quotes ("") to define a string.

2. With template literals, we can use both single and double quotes inside a string.

3. Template literals allows multiline strings.

4. Template literals provide an easy way to interpolate variables and expressions into strings.

5. Template literals allow variables, expressions in strings.
